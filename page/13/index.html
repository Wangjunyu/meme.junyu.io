mkdocs

实现把所有的需要使用的文档利用mkdocs做了整合，体验如下：

  • 站点不要太多，比如工作、生活、输出，也就三四个大分类就够了。mkdocs中对细分分类做整合的能力很强，有一级、二级、三级目录都可用，搞得太零碎会影响自己的正常使用。
  • 需要熟悉mkdocs的基本配置和插件的使用，不懂的问AI，比如：利用软连接实现把别的位置的目录，整合到当前的根目录下，实现一个位置的文件在多个mkdocs中展示。
  • 生成网页时会遍历docs下的文件，如果本地使用，建议不要把太多非md或者相关文件放到docs下，会极大的降低生成效率，我还出现过直接把电脑空间干不足的情况。慎用…
  • 时刻谨记，避免进入时间黑洞，先想好自己想要的效果，用最简单有效的方式去实现。不要陷入到无尽的技术细节中无法自拔…

mkdocs

初步实现了本地电脑的文件文档化，即本地的文件夹要么是mkdocs的文档，要么是类似hugo的文档，启动之后，浏览器端查看就是一个网站。各种文件成为附件。

这里有一个问题，有一些站点的内容，是可以全部开放的，本来就是希望分享出去。

但是一些内容是比较私人的，如何在一个主题下，既有私人的内容，不上传git，又有公开的内容呢？

# 在.gitignore中添加不想上传的文档路径
docs/private/*
docs/draft/*

# mkdocs-local.yml (本地开发用)
nav:
  - Home: index.md
  - Public: public.md
  - Private: private.md

# mkdocs.yml (git版本)
nav:
  - Home: index.md
  - Public: public.md
  
# 只需要在本地增加目录，同时单独配置一个yml文件

# 使用完整配置
mkdocs serve -f mkdocs-local.yml

# 或使用默认配置
mkdocs serve
